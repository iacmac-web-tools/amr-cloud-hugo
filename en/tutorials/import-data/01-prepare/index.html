Data preparation

The “Golden Rules” for the source data

One of the key points of working with the AMRcloud platform is the structuring of your source data.

The main principles of data organization can be formulated as the following “Golden Rules”:

  1. Each isolate is represented by a separate row in the source datatable.
  2. There are five mandatory columns in the source datatable (Isolate ID, Species Name, Organism Group, Date, AST Data).
  3. There always should be one of the three suffixes after the generic name of antimicrobial drug («_mic», «_dd» or «_sir»).
  4. All other additional columns with metadata in the source datatable are NOT mandatory (geographical data, resistance markers, etc.).

Example Data

The basic requirements for the source data table are listed on the Read Me tab of the data import page. At the bottom of the page there is a link to download an example data table that can be used for the original user’s data formatting.

Basic Requirements

To work with the AMRcloud it is necessary to convert the data to a single format.

  1. The data table should be flat. It means that it consists of columns and rows with no additional separators, no hierarchical structures.
  2. The first row of the data table should contain the headings of all columns.

  3. The data table should contain 5 types of mandatory columns:

    • Isolate Identifier (ID_isolate)
    • Organism/Species Name
    • Organism Group
    • Date
    • AST Data

Isolate ID

Non-unique numbers can be used as the isolate ID. The identifier can be either alphabetic or numerical.

Organism/Species Name

For the automatic interpretation Species name must be full. Abbreviations in the species name are not allowed.

Right:

Staphylococcus aureus, Streptococcus pneumoniae

Wrong:

S. aureus, S. pneumoniae

Organism Group

The Organism Group can be shortened, but it should be identical throughout the column.

Right:

Enterobacterales, Staphyl, Enterobacterales, Staphyl (all names are standardized)

Wrong:

Enterobac, Staph, Staphyl, Entbac, Staphylococ (names are not standardized)

Date

The import system will automatically interpret any standard date format. The main condition is the universality of the date format throughout the column.

Right:

12.08.2018, 20.05.2017, 16.03.2014 (unified format DD.MM.YYYY)

Wrong:

13.08.2018, 20/05/19, 03/16/2014 (non-unified format DD.MM.YYYY, DD/MM/YYYY, MM/DD/YYYY)

AST Data

Possible AST data formats:

  • Minimum Inhibitory Concentrations (MICs)
  • Disk Diffusion Inhibition Zone Diameters
  • Susceptibility Categories (S/I/R)

Antimicrobial Suffixes

The header of the column with the results of the antibiotic susceptibility testing must comply with the following rules:

  1. The name of the antimicrobial agent should be full without any reductions, written in English. The list of the legitimate antibiotic names can be downloaded by the link at the bottom of the page.
  2. Every generic ‘Antibiotic’ name shouled be followed by one of the three suffixes with underline symbol:
Minimum Inhibitory Concentrations_mic
Disk Diffusion Inhibition Zone Diameters_dd
Susceptibility Categories_sir

Example:

tobramycin_sir, tetracycline_sir, amoxicillin-clavulanic acid_mic, daptomycin_dd

Note

Antimicrobial agents with special interpretation criterias (user’s S/I/R categories) and/or ECOFF criterias can be recorded as follows:

meropenem_ecoff_sir, moxifloxacin_clsi_sir, marbofloxacin_clsi_sir

IMPORTANT!

  • Minimum inhibitory concentration values

    • MIC values are measured in mg/l.
    • Values must be numerical.
    • Non-numerical values containing expressions <= ... or > ... or >= ... would be converted into numerical.
    • Symbols <= and >= standing before the numbers are ignored.
    • Numbers standing after > symbol would be multiplied by 2.

  • Disk Diffusion Inhibition Zone Diameters

    • Diameter values measured in mm.
    • Values could be integers greater than or equal to 6.

  • Susceptibility Categories

    • Allowed values: “S”, “I”, “R”.
    • Every isolate may have only one susceptibility category.

Right:

S, S, R (one susceptibility category for every isolate)

Wrong:

S+I, S+I, I+R (more than one susceptibility category for every isolate)

Additional Columns

Additional Columns — filling in such columns is not mandatory.

  1. Spatial Information
    • geographical objects (city, region, country, etc.)
    • latitude
    • longitude
  2. Metadata (textual data)
  3. Metadata (numerical data)
  4. Resistance Markers (resistance mechanisms, genes or mutations, etc.)

Spatial Information

Geographical objects may include the name of a city, region, country. User can also specify more detailed address (e.g. specific hospital wing). Latitude and Longitude can be changed manually if automatic determination of the objects’ coordinates did not work.

The main requirements and recommendations for the geographical data are presented in the “Step 3: Geocoding” stage.

Metadata (textual data)

User can select a maximum of 10 parameter columns containing textual metadata. For example, “Clinic Material Localisation”, “Hospital department”, etc.

It is recommended to include the column with geographical objects names in here. Individual patient’ characteristics may also be included in here (e.g. “Febrilic Fever” column). Recommended values: “Yes”, “No”, “Not applicable”.

Metadata (numerical data)

User can select only one continuous numeric variable such as “Patient’s age”, “weight”, etc.

Resistance Markers

  • Total number of resistance markers is unlimited.
  • User may create his own “group of markers”.
  • Resistance markers may include resistance mechanisms, genes or mutations, and other data that can be correlated with phenotypic resistance data .

The following notation system is recommended:

Investigation of the specific genetic or phenotypic markers…Value(s)
…was conducted with positive test result“OXA-48”, “KPC”, “MRSA”, “CTX-M-15” etc.
…was conducted with negative test result“Negative”
…was not conducted“Not defined”
…is not applicable for the microorganismempty cell
