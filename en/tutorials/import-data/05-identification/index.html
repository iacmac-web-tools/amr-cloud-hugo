Step 2 - Set Fields

At this stage for correct work of all analytical components of the AMRcloud platform user should correctly correlate the names of eight standard system categories with the columns from the uploaded data table.

On the left side of the screen there are standard categories (for example, «Isolate ID», «Organism Group(s)») with basic information and requirements. On the right side of the screen there are panels with drop-down lists of table column headers, with which the user correlates system categories.

Isolate ID

  • This is a mandatory category.
  • Isolate ID can be any unique identifier that doesn’t include special characters (?,!,%$#><)

Organism Group(s)

  • This is a desirable category.
  • Grouping of organisms (species) is useful for analyzing combined prevalence and AST data of several species.
  • Use any name(s) for organism group(s), taxonomic (e.g. Enterobacterales) or non-taxonomic (e.g. Anaerobes, Gramnegative).

Organism/Species Names

  • This is a mandatory category.
  • Use Latinized binomial names acoording to official nomenclature (e.g. Escherichia coli).
  • Proper genus and species names are automatically recognized to apply specific breakpoints and define susceptibility categories.

Right:

Klebsiella pneumoniae, Acinetobacter baumannii, Pseudomonas aeruginosa (no reductions, corresponds to the official nomenclature)

Wrong:

K. pneumoniae, A. baumannii, P. aeruginosa (reductions are used)

After selecting the appropriate column name from the drop-down list, user can press the «Check Species Names» button for automatic checking.

After pressing the «Check Species Names» button a small table will appear with the names that fails the verification process (i.e. the names of microorganisms which were not recognized by the system). Then user should look through this table carefully:

  • If there are errors in some names (for example, a missing letter), user should correct the errors right in the editor window by double-clicking the necessary cell with the left mouse button and pressing the «Correct species names» button.
  • If there are no spelling errors, it means that there are no microorganisms with such names in the system. Such microorganisms are not excluded from the general data — analysis charts will also be built for them — only they will be colored not according to susceptibility categories, but in a solid gray color. If user works with his own S/I/R susceptibility categories, there will be no problem in identifying microorganism species.

If interpretation criteria for particular microorganism species have not been found, the system will try to apply criteria for the appropriate microorganism group.

Example. If Salmonella typhi criterias are not found, then Salmonella spp. criterias would be applied.

Date

  • This is a mandatory category.
  • Records without a date or with a varying date format will not be imported.

If all the dates are arranged correctly and uniformly, user (after selecting the corresponding header from the drop-down list) will see the time scale «Date Range Privew». If the date format is not recognized with the Auto option, user should try to select one of the alternative formats: - 2018-12-31 (i.e. year-month-day) - 2018-31-12 (i.e. year-day-month) - 31-12-2018 (i.e. day-month-year) - 12-31-2018 (i.e. month-day-year)

Once user has selected the date format, he should check the box «Check if you see the time scale?».

Spatial Information

  • This is not a mandatory category.
  • There are two types of spatial information: «Geolocation» and «Longitude/Latitude Coordinates».

Geolocations can include regions, such as countries and states, natural areas, cities and smaller locality types. The system will attempt to identify geolocations and find respective geographic coordinates (latitude and longitude) automatically. Optionally, user can check the box and select the columns containg the Longitude and Latitude values.

Please note that only the first 10 rows of the corresponding column are displayed to the user for preview.

Numeric Parameter

  • This is not a mandatory category.
  • As a numeric parameter, user can upload one of any continuous numeric variable, for example Age, Weight, etc.

User should check the corresponding box on the right side of the screen and select the table header form the drop-down list.

Please note that only the first 10 rows of the corresponding column are displayed to the user for preview.

Resistance Markers

  • This is not a mandatory category.
  • The total number of columns of this type is unlimited.

If there are column(s) containing information about Resistance Markers (resistance mechanisms, genes or mutations) that can be correlated with phenotypic resistance data, check the box and then select the respective column(s). Column header(s) will be used as name(s) of the Marker Groups.

Additional Parameters

  • This is not a mandatory category.
  • A maximum of 10 Parameter Columns can be selected.
  • Column titles will be used as names of the parameters.
  • Drag selected parameters to set their priority from 1 to 10.

Completing Step Two

To go to the next stage of the data import user should press the «Validate Step Two» button. It is recommended to review and double-check all the points of the current stage before saving.
