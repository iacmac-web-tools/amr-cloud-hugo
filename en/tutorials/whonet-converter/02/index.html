2. AMRcloud convert

To convert file from step 1 of this tutorial, follow the link https://public.amrcloud.net/whonet.

Upload file

Press the Choose file button and select your export file from step 1.

The file upload will start automatically. When the upload is complete, the converter will check the contents of the file, try to read it, and display a preview on the screen.

The export file must meet the following requirements:

  • Maximum file size 10 MB;
  • Separator Tab;
  • All column names must be unique.

The standard file from step 1 meets these requirements. If the upload is successful, you will see the following.

If the file was not read correctly, you will see the error message:

In this case, the error is that the column named FOX_ND30 occurs more than once in the table.

If you see any errors after file upload, it means that further conversion is impossible.

Convert file

The main idea of the transformation process is to replace WHONET codes with another codes that the AMRcloud understands.

Replacements are made for:

  • Antibiotis (columns like CZX_ND30, VAN_NM, VAN_NE, where the first part is recognized as an antibiotic, the second as a testing method)
  • Microorganisms (replacing ORGANISM column values, and adding ORGANISM_GROUP column)
  • Clinic materials (replacing SPEC_TYPE column values)

To convert uploaded file press the Convert button. At the end of the transformation process, a report on the performed manipulations will appear on the page. The example of such a report is shown on figure 4.

First, warnings are displayed - these are some features identified during the conversion process that you should pay attention to. In this case, there is only one warning and it says that a new column ORGANISM_GROUP has been added and automatically filled in the original table.

This is requirement field, because AMRcloud uses two-level hierarchy for microorganisms.

Message Comment
Converted table has multiple columns with antibiotic. Columns will be enumerated This message is displayed when two or more antibiotics receives identical replacements as a result of recoding. For example, in our case, the file had two columns AMK_ND30 and AMK_ND20. Each of them should be replaced with amikacin_dd, but to preserve the test data and the uniqueness of the column names, a number is appended to the new names. So AMK_ND30 becomes amikacin_dd, and AMK_ND20 becomes amikacin_dd_1.
No replacement found for … This message is displayed when any value to be replaced was not found in the dictionary. In this case, the original value is preserved. You should pay attention to this value and, if necessary, make a replacement yourself. For example, in our case, no substitutions were found for the antibiotic code AMM_NE and material tt.
Table doesn’t has ORGANISM column This message is displayed when the ORGANISM column was not found in the table. This situation is not a critical error, but it is a signal that the converted file cannot be uploaded to AMRcloud without information about microorganisms.

After the warnings, there are replacement tables. You can evaluate the correctness of the replacements made and, if necessary, correct the incorrect ones yourself.

Attention! How the name of the replacement antibiotic is formed:

  • Column title is split before and after the underscore;
  • The first part is replaced according to the dictionary;
  • The content of the antibiotic column is analyzed. If at least one value of S, I or R is found, the code _sir is appended;
  • If the antibiotic column does not contain S, I or R values, the second part of the name of the original antibiotic column is used and the following codes are appended:
    • ND – _dd
    • NM – _mic
    • NE – _etest_mic
  • If it was not possible to determine the test method by analyzing the second part of the original name of the antibiotic column, then the code _mic is added.

Download file

To download converted file, press the Download button. It is assumed that the file is already ready for uploading to AMRcloud and does not require additional editing (except for the situation with Warning: Table doesn’t has ORGANISM column).

How to upload the file to AMRcloud, you can found in the tutorial.
